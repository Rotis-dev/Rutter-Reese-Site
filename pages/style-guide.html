Style Guide (Ops Docs)

Short rules that make docs consistent, auditable, and fast to use under pressure.

Document anatomy

  • Purpose (one sentence)
  • Audience (who uses this)
  • Prerequisites (access/tools/permissions)
  • Safety & warnings (what can go wrong)
  • Steps (numbered, one action per step)
  • Decision points (IF/THEN with next step references)
  • Escalation (who/when/how)
  • Verification (how you know it worked)
  • Records (what to log, where)

Language rules

  • Use imperative voice: “Open…”, “Verify…”, “Record…”
  • One step = one action. Avoid “and” in steps.
  • Use consistent terms. Add a glossary if needed.
  • Put numbers in numerals (3, 10) when they are parameters.

Warnings & callouts

  • WARNING: Safety risk or equipment damage.
  • CAUTION: Procedure risk (wrong command, wrong state).
  • NOTE: Helpful context that doesn’t change the steps.

Formatting rules

  • Headings: H1 doc title; H2 major sections; H3 subsections.
  • Use tables only for quick-reference, not long explanations.
  • Prefer checklists for repeatable verification steps.
