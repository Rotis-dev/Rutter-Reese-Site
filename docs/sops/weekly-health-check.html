SOP — Weekly System Health Check (Generic Example)

Version: v1.1 — 2026-01-01 · Sanitized portfolio sample — not operational guidance.

Purpose

Perform a consistent weekly health check and produce a record that supports trend detection, follow-ups, and accountability.

Audience

Ops/Support staff responsible for basic monitoring and recordkeeping.

Scope

  • Review monitoring health and data freshness
  • Review alarms/trends for last 7 days
  • Confirm comms uptime and note outages
  • Confirm power/backup indicators (if applicable)

Inputs

  • Monitoring dashboard(s)
  • Status page (if used)
  • Weekly log (ticket/KB/spreadsheet)

Outputs

  • Completed weekly entry
  • Top issues + follow-ups with owners
  • Notes on outages/abnormalities

Prerequisites

  • Access to monitoring dashboard(s) and system status page
  • Access to the weekly log (ticket, spreadsheet, or KB page)
  • Ability to assign an owner (name/role) for follow-ups

Procedure

  1. Open the weekly log and create a new entry for this week.
  2. Verify telemetry freshness (last update within the expected window).
  3. Review critical alarms for the last 7 days; list the top 3 recurring alarm types.
  4. Check communications uptime; record outages or notable instability.
  5. Check power/backup health indicators (if applicable); record status and any anomalies.
  6. Document open follow-ups and assign an owner (name/role) and due date.
Weekly log template (copy/paste)
  • Week of: YYYY-MM-DD
  • Telemetry freshness: OK / Not OK (notes)
  • Top recurring alarms (3): 1) … 2) … 3) …
  • Comms uptime notes: …
  • Power/backup notes: …
  • Follow-ups: (Owner, due date, link/ticket)
  • Summary (3–5 lines): …

Quality checks

  • All required fields completed (no “TBD”).
  • Any abnormality has at least one follow-up item with an owner and due date.
  • Summary is clear enough that someone else can understand status in 60 seconds.

Records

  • Weekly log entry link or ID
  • Top recurring alarms list
  • Follow-up list (owner + due date)
